Hint: Think about how the values 'block', 'inline', and 'inline-block' change the element's behavior in layout.
Answer: The 'display' property in CSS defines how an element is displayed. By default, block-level elements (e.g., <div>) take up the full width, while inline elements (e.g., <span>) only take up as much space as needed. Using 'display: block', you can turn an inline element into a block-level element, and vice versa with 'display: inline'. 'display: inline-block' allows an element to maintain its inline status but behave like a block element.
Hint: Focus on how each positioning type affects the element's position relative to its containing block or viewport.
Answer: 'absolute' positioning removes the element from the normal document flow and positions it relative to its closest positioned ancestor. 'relative' positioning moves an element relative to its normal position, without removing it from the flow. 'fixed' positioning removes the element from the document flow and fixes it relative to the viewport, meaning it stays in place as the user scrolls.
Hint: Consider how elements move in relation to other floated elements and how the 'clear' property is used in conjunction.
Answer: The 'float' property allows elements to move to the left or right of their containing element, allowing other content to wrap around it. For example, floating an image to the left with 'float: left;' will cause text to wrap around the image on the right. The 'clear' property is used to prevent elements from wrapping around floated elements.
